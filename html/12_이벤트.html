이벤트(Event) 

        * 이벤트 : 동작, 행위, (행사)
        -> 브라우저에서 할 수 있는 동작
        ex) click, keydown, keyup, keypress, keyup
            mouseover, mouseon, mouseout, mouseleave,
            drag, change, sbmit, ....
            이벤트 종류 참고
        

        * 이벤트 리스너(EventListner)
        - > 브라우저에서 이벤트가 감지될 때 가지 대기하다가
            감지가 되었을 때 연결된 기능(함수)을 수행하는 것
        - > 이벤트 리스너는 "on이벤트명" 형식으로 작성됨
            ex) onclick, onchange, onsubmission, onkeyup ... 

        * 이벤트 핸들러(Event Handler)
            > 이벤트 리스너에 연결된 기능(함수)으로
              이벤트가 감지되었을 때 수행할 내용을 작성한 함수(function)

인라인 이벤트 모델(Inline Event Model)

        - HTML 요소에 이벤트리스너 = "이벤트핸들러"
          형식을 작성하는 방법

        - 하나의 요소에 동일한 이벤트핸들러를 중복해서 작성 불가
          이전에 작성된 이벤트핸들러를 새로운 이벤트핸들러가 덮어쓰기함
          
        * 최근에는 많이 쓰진 않지만 종종 사용됨

        * 인라인 이벤트 모델 작성 시 this
         -> 이벤트가 발생한 현재 요소

고전 이벤트 모델

        - HTML 요소에 이벤트 관련 코드를 작성하지 않고
          JS 에서 DOM 객체에 직접 이벤트 관련 코드 작성 방법 

        - 단점 : 하나의 요소에 동일한 이벤트핸들러 중복작성 불가

표준 이벤트 모델(요소.addEventListner())

        - W3C (웹 표준 지정 단체)에서
          공식적으로 지정한 이벤트 모델

        - 장점 : 한 요소에 중복되는 이벤트리스너를 만들어 넣어
                 여러 이벤트 핸들러를 추가할 수 있다
                 (고전 이벤트 모델 단점을 해결)
